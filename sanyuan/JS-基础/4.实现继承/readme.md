2. 借助原型链
  有一个问题 创建多个实例，改变其中一个实例，其他的也会改变，因为是指向同一个原型
3. 将前面两种结合
  缺点：Parent3的构造函数会多执行了一次（Child3.prototype = new Parent3();）

  父类原型对象直接给到子类，父类构造函数只执行一次，而且父类属性和方法均能访问
